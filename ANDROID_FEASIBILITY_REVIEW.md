## 🚨 주요 기술적 도전 과제

### 1. 🔴 매우 어려운 부분 (Critical)

#### TipTap 에디터 구현 (난이도: ⭐⭐⭐⭐⭐)
**문제점:**
- TipTap은 웹 전용 ProseMirror 기반 에디터
- 30+ 확장 기능 (Callout, 코드 하이라이팅, 수식, YouTube 임베드 등)
- 드래그 앤 드롭, 복잡한 DOM 조작

**해결 방안:**
- **React Native**: `react-native-webview`로 에디터만 웹뷰 임베딩 (하이브리드)
- **네이티브**: Aztec Editor (WordPress) 또는 자체 구현 (6개월+ 소요)
- **실용적 접근**: 모바일에서는 읽기 전용 + 간소화된 편집 UI 제공

#### 코드 신택스 하이라이팅 (난이도: ⭐⭐⭐⭐☆)
**문제점:**
- Lowlight (30+ 언어 지원)
- 웹 기반 하이라이팅 라이브러리

**해결 방안:**
- **React Native**: `react-syntax-highlighter` 사용
- **네이티브**: Highlight.js 또는 Prism 라이브러리 연동

#### 수학 수식 렌더링 (난이도: ⭐⭐⭐⭐☆)
**문제점:**
- KaTeX 웹 라이브러리 의존

**해결 방안:**
- **React Native**: `react-native-math-view` 또는 WebView
- **네이티브**: MathJax Android 라이브러리

### 2. 🟡 중간 난이도 부분 (Moderate)

#### 이미지 업로드 및 관리 (난이도: ⭐⭐⭐☆☆)
**문제점:**
- 드래그 앤 드롭 (모바일 미지원)
- OCI Object Storage 연동
- Sharp 이미지 처리

**해결 방안:**
- 갤러리/카메라 접근 API 사용
- 파일 선택기로 대체
- 백엔드 API는 그대로 사용 가능

#### 인증 시스템 (난이도: ⭐⭐☆☆☆)
**문제점:**
- Google OAuth 웹 플로우
- JWT 토큰 관리

**해결 방안:**
- **React Native**: `@react-native-google-signin/google-signin`
- **네이티브**: Google Sign-In SDK
- JWT는 SecureStore/Keychain에 저장

#### UI/UX 반응형 디자인 (난이도: ⭐⭐⭐☆☆)
**문제점:**
- Tailwind CSS 웹 전용
- 데스크톱/모바일 레이아웃 차이

**해결 방안:**
- **React Native**: NativeWind (Tailwind for RN)
- 모바일 최적화 레이아웃 재설계

### 3. 🟢 쉬운 부분 (Easy)

#### REST API 통신 (난이도: ⭐☆☆☆☆)

### 📱 전략 2: PWA (가장 빠른 대안 ⭐⭐⭐⭐☆)

#### 장점
- **개발 기간**: 1주일 이내
- **코드 변경 최소화**: Service Worker + Manifest 추가만
- **비용**: 거의 $0 (기존 인프라 사용)
- **유지보수**: 웹과 동일

#### 구현 단계
```bash
# 1. Vite PWA 플러그인 설치
pnpm add -D vite-plugin-pwa

# 2. vite.config.ts 수정
import { VitePWA } from 'vite-plugin-pwa'

export default {
  plugins: [
    VitePWA({
      registerType: 'autoUpdate',
      manifest: {
        name: 'JaeHyeong Tech Blog',
        short_name: 'JHTech',
        theme_color: '#ffffff',
        icons: [...]
      },
      workbox: {
        // 오프라인 캐싱 전략
      }
    })
  ]
}

# 3. 모바일 최적화 CSS 추가
# 4. 테스트 및 배포
```

#### PWA 기능
- ✅ 홈 화면 추가 (아이콘)
- ✅ 오프라인 모드 (Service Worker)
- ✅ 푸시 알림 (Web Push API)
- ✅ 앱처럼 실행 (전체화면)
- ⚠️ 네이티브 API 제한 (카메라, 파일시스템 등)

#### 제한사항
- iOS Safari 제약사항:
  - 홈 화면 추가 수동 (설치 프롬프트 없음)
  - 푸시 알림 iOS 16.4+ 지원
  - 백그라운드 동기화 미지원
- 네이티브 느낌 부족
- 앱스토어 배포 불가 (사용자가 웹에서 설치)

- 웹 + 앱 병행 유지보수 필요
- 버그 발생 시 플랫폼별 테스트 필요
- **해결책**: React Native로 코드 공유 최대화

### 3. 앱스토어 정책
- Google Play: 콘텐츠 정책 준수 필요
- Apple App Store: 리뷰 프로세스 (1-2주)
- **주의**: 웹뷰 래퍼만 있는 앱은 거절될 수 있음

### 4. 성능 이슈
- 복잡한 포스트 렌더링 시 메모리 사용량 증가
- 이미지 다수 포함 시 로딩 최적화 필요
- **해결책**: 가상 스크롤 (FlatList), 이미지 레이지 로딩

## 💬 질문/답변

**Q: 안드로이드만 만들 수 있나요?**
A: React Native 사용 시 iOS도 90% 코드 공유로 동시 개발 가능합니다. Flutter도 동일합니다.

**Q: TipTap 에디터를 그대로 써야 하나요?**
A: 모바일에서는 간소화된 마크다운 에디터를 권장합니다. 복잡한 편집은 웹에서 하는 것이 UX상 더 좋습니다.

**Q: 기존 웹사이트에 영향이 있나요?**
A: 전혀 없습니다. 백엔드 API는 그대로 사용하고, 앱은 별도 클라이언트입니다.

**Q: 앱스토어 수익화가 가능한가요?**
A: 가능합니다. 광고, 인앱 구매, 구독 등 다양한 수익화 모델을 적용할 수 있습니다.

**Q: 푸시 알림으로 새 글 알림을 보낼 수 있나요?**
A: React Native/Flutter에서는 FCM으로 가능하고, PWA에서도 Web Push API로 가능합니다 (iOS 16.4+).
